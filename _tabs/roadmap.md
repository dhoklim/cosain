이 로드맵은 **C++ 입문부터 BOJ Diamond/Ruby와 ICPC World Finals 수준의 주제까지** 이어지는 PS/CP 학습 지도입니다. 알고리즘 이름을 많이 아는 것보다 문제의 제약을 읽고, 필요한 복잡도를 계산하고, 풀이를 증명하고, 정확하게 구현하는 능력을 목표로 합니다.

> BOJ 난이도는 학습 순서를 설명하기 위한 대략적인 범위입니다. 같은 알고리즘도 모델링과 구현 난도에 따라 티어가 크게 달라집니다.
{: .prompt-info }

## 로드맵 사용법

1. **0~6단계는 순서대로** 학습합니다. 뒤 단계에서 계속 사용하는 공통 기반입니다.
2. **7~12단계는 트랙별로 병행**합니다. 한 트랙을 모두 끝내고 다음 트랙으로 넘어갈 필요는 없습니다.
3. 개념을 읽은 뒤 최소 3문제를 풉니다. 첫 문제는 구현, 둘째는 변형, 셋째는 여러 개념이 섞인 문제로 고릅니다.
4. 정답을 맞힌 뒤 시간복잡도, 정당성, 실수한 지점을 짧게 기록합니다.
5. 완료 기준을 만족하지 못하면 다음 단계로 서두르지 말고 취약한 구현을 반복합니다.

### 전체 흐름

| 구간 | 권장 난이도 | 도달 목표 |
|------|-------------|-----------|
| 0단계 | 입문 ~ Bronze | C++로 제출하고 오류를 스스로 찾는다 |
| 1~2단계 | Bronze ~ Silver | 자료구조와 완전탐색을 직접 구현한다 |
| 3~6단계 | Silver ~ Gold | 전형 문제에서 알고리즘을 스스로 선택한다 |
| 7~10단계 | Gold ~ Platinum | 그래프·트리·DP·문자열 문제를 구조화한다 |
| 11~12단계 | Platinum ~ Diamond/Ruby | 수학·기하·오프라인 고급 기법을 결합한다 |
| 13단계 | Platinum ~ Ruby | 처음 보는 문제를 증명하고 팀 단위로 해결한다 |

## 0단계 — C++와 PS 환경

**선수 지식:** 없음

### 학습 주제

- 표준 입출력, 조건문, 반복문, 함수
- 배열, `string`, 참조, 구조체, 클래스의 최소 문법
- `vector`, `pair`, 반복자, 람다, 범위 기반 반복문
- 정수 범위와 오버플로: `int`, `long long`, `__int128`
- 시간복잡도와 공간복잡도, 입력 크기로 가능한 연산량 추정
- 컴파일 오류, 런타임 오류, 시간 초과, 메모리 초과의 구분
- 빠른 입출력과 C++17 제출 환경

**완료 기준:** 빈 파일에서 입출력 템플릿을 작성하고, $O(N)$·$O(N \log N)$·$O(N^2)$ 중 입력 제한에 맞는 복잡도를 고를 수 있다.

**관문 문제:** BOJ 1000, 10871, 10951, 11720

**관련 글:** 예정

**선수 지식:** 0단계

- 사용자 정의 정렬 기준과 구조체 비교

**완료 기준:** 문제의 연산을 보고 필요한 자료구조를 선택하고, STL을 사용한 구현의 복잡도를 설명할 수 있다.

**관문 문제:** BOJ 10828, 10845, 10866, 1927

**관련 글:** 예정

## 6단계 — 그래프 탐색과 동적 프로그래밍 입문

**선수 지식:** 1~5단계

## 13단계 — 최상위 통합 문제 해결

최상위 문제는 새로운 알고리즘 하나보다 여러 분야의 결합에서 나오는 경우가 많습니다.

### 자주 등장하는 결합

- 트리 + HLD + lazy segment tree
- DP + Convex Hull Trick 또는 분할 정복 최적화
- 문자열 오토마톤 + DP 또는 행렬 거듭제곱
- 최대 유량 + 이분 탐색 + 최소 컷 모델링
- 오프라인 동적 연결성 + DSU rollback
- 계산 기하 + 스위핑 + 균형 자료구조
- 다항식 합성곱 + 조합론 + 생성함수

### 풀이 절차

1. 입력 크기에서 허용되는 목표 복잡도를 계산한다.
2. 작은 예시와 완전탐색으로 구조를 관찰한다.
3. 상태, 정점, 간선, 질의 중 어떤 모델이 적합한지 정한다.
4. 필요한 알고리즘의 전제 조건을 하나씩 검증한다.
5. 정당성 증명과 복잡도 분석을 코드보다 먼저 쓴다.
6. 단순한 정답 코드와 무작위 테스트를 만들어 스트레스 테스트한다.

**완료 기준:** 처음 보는 문제에서 60~90분 동안 관찰과 모델링을 지속하고, 실패한 접근도 반례와 함께 설명할 수 있다.

## 대회 실전 훈련

알고리즘 학습과 대회 성적은 같은 일이 아닙니다. 아래 훈련을 별도로 반복합니다.

### 개인 훈련 루틴

- **매일:** 취약 유형 1문제와 복습 문제 1문제
- **매주:** 2~3시간 가상 대회 1회
- **대회 후:** 못 푼 문제를 난이도순이 아니라 배울 가치순으로 업솔빙
- **월 1회:** 템플릿을 빈 파일에서 다시 작성하고 불필요한 코드를 제거
- **상시:** WA, TLE, RE의 원인과 재발 방지 규칙 기록

### ICPC 팀 훈련

- 시작 10~15분 동안 전 문제를 읽고 난도와 구현량을 추정한다.
- 문제마다 풀이 담당자와 검수 담당자를 구분한다.
- 증명되지 않은 아이디어는 코딩 전에 반례를 함께 찾는다.
- 한 컴퓨터를 쓰는 상황을 가정해 종이 디버깅과 구현 대기열을 연습한다.
- 페널티, 제출 순서, 풍선 정보보다 팀의 확실한 풀이를 우선한다.
